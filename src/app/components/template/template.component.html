Template

Text interpolation

Inject dynamic string values into your templates with text interpolation.

Text interpolation uses double curly braces {{ ... }} as delimiters.

Nickname: {{ nickname }}

Template expressions

The text between the double curly braces is a template expression . Angular evaluates the template expression, then converts it to a string.

2 * 5 = {{ 2 * 5 }}

{{ true }} && {{ false }} = {{ true && false }}

You can also call methods from the component:

{{ leet[0] }} join {{ leet[1] }} equals {{ leetJoin() }}

Template statements/event binding

Template statements/event binding are properties or methods that you can use in your templates to respond to user events.

Syntax follows as: (event)="statement"

Pipes

Pipes are simple functions used in template expressions to transform input values.

Complete list of built in pipes:

  • {{ builtInPipes }}

{{ today }}

{{ today | date }}

Pipes with parameters

{{ num1 | currency }}

{{ num1 | currency: 'EUR' }}

{{ num1 | currency: 'JPY': 'Japanese Yen ' }}

Chained pipes

{{ today | date | uppercase }}

Property binding

Property binding sets values for properties of HTML elements or directives.

Text interpolation vs Property binding

From Angular:

You can use either form when rendering data values as strings, though interpolation is preferable for readability. However, when setting an element property to a non-string data value, you must use property binding.

Template reference variables

Template reference variables can use data from one part of a template in another part of the template. You use the hash symbol (#) to create a template variable.
